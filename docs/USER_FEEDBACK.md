# PhDapp — Trial Feedback

Thanks for helping test PhDapp. Please **use it for your normal work** for a
couple of weeks and jot down **anything** — things that broke, things that
confused you, things you wish it did. Nothing is too small or too silly.

You don't need to be technical. A rough note is far better than no note.

---

## How to take part

1. Sign in at **https://phdapp.vercel.app** with the Google account you were
   added with. (If you see "Access blocked", your email isn't on the list
   yet — tell Rafa.)
2. Use it normally — your tasks, calendar, chat, reading, etc.
3. Whenever something is wrong/confusing/missing, **add an entry below** in the
   matching section, using the template. Don't edit other people's entries —
   just add yours underneath.
4. Rafa collects this periodically and the issues get fixed.

### What makes a report useful (30 seconds)

- **Where** you were: the page name, and if you can, the web address in the
  browser bar (e.g. `…/students/abc123` or `…/kanban`).
- **What you did**, step by step (even "clicked Save").
- **What happened** vs **what you expected**.
- A **screenshot** if you can (Mac: Shift+Cmd+4; Windows: Win+Shift+S) — save
  it and note the filename, or send it to Rafa with the entry number.
- Your **role** (student / supervisor / committee / advisor / admin) and rough
  **date/time** — helps reproduce it.

### How bad is it? (pick one)

- **BLOCKER** — I couldn't do the thing at all, or I think I lost data.
- **BROKEN** — wrong/buggy, but I found a workaround.
- **CONFUSING** — it works, but it's unclear, awkward, or looks off.
- **IDEA** — not a bug; something that would make it better.

### Copy this template for each item

```
### [#] Short title
- Who: <your name> — <role>
- When: <date, rough time>
- Where: <page / web address>
- Severity: BLOCKER | BROKEN | CONFUSING | IDEA
- What I did: 1) … 2) … 3) …
- What happened: …
- What I expected: …
- Screenshot: <filename, or "sent to Rafa", or "none">
```

### Example (so you can see the format)

```
### [12] Calendar event shows as deleted right after I create it
- Who: Ada — student
- When: 17 May, ~10:30
- Where: Calendar (…/calendar), Month view
- Severity: BROKEN
- What I did: 1) New event 2) filled title + time 3) Save
- What happened: it appeared, then a few seconds later showed crossed-out like it was deleted
- What I expected: it to just stay there
- Screenshot: calendar-ghost.png (sent to Rafa)
```

Number your items however you like (initials + number is fine, e.g. `[AL-1]`)
so two people don't clash.
